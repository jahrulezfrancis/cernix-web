## Immediate outcome

Connect the existing queued `snapshot_jobs` record to the existing immutable repository snapshot service:

```text
queued
→ leased
→ snapshot service
→ succeeded + snapshotting → planning
```

with:

```text
transient failure → retry_wait
terminal/exhausted failure → failed + investigation failed
worker crash → lease expiry → safe reclamation
```

## Required implementation areas

- New migration after all merged migrations.
- Durable job status/lease/retry fields.
- Durable attempt-history table.
- `FOR UPDATE SKIP LOCKED` claim repository.
- Database-time lease ownership.
- UUID fencing token per attempt.
- Heartbeat and expired-lease recovery.
- Central retry classifier and persisted exponential backoff.
- Worker `runOnce` and loop modes.
- Separate CLI process and graceful shutdown.
- Atomic success transition to `planning`.
- Atomic terminal failure transition to `failed`.
- Idempotent replay when snapshot already exists.
- Safe attempt/job logging.
- Offline unit tests and guarded PostgreSQL concurrency tests.
